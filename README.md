wt2c 
(web throw to catch)
=====

__look at notes for crucial information!__

*wt2c* is an online service written in JavaScript and HTML, acting as a quick and easy storage space for a multitude of HTML, CSS and JS* files. The aim of wt2c is to reduce website storage taken up by these documents by compressing* or translating them. Then, when said documents are requested wt2c will decompress* or retranslate them for display on the samm-r.github.io/catch/#/... output site. wt2c uses GitHub for it's machinery.

### Keys
key terms wt2c uses are throw and catch, throwing meaning to send or register your files amongst wt2c's directory and catching meaning to receive said files to output them on the wt2c (github) site. Weird terms but think of it as a digitalised verison of passes.

### Usage
to use wt2c, you must first register* you and your files on the /throw/ directory of the wt2c URL, and when verified you can insert an iframe `<iframe src="https://samm-r.github.io/catch/#/...'` into your index HTML file, but make sure to replace the ellipses in the tag with the name-key you received after registration.

### Notes
wt2c, though functional is still in development and an actual complete release may be whats ahead in the future. 
the service cannot register any files thrown yet, but once developed to a functional degree will be released 
to the public.

wt2c is at a basic level of functionality too, with only being able to accept basic 1-2 character tags like `<p>` or `<h1>`. 

1*: html docs supported for now.

2*: compression not available yet

3*: throw service not available, registration not available.
